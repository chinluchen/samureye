# Samureye 產品與技術路線圖

## 產品定位

Samureye 是一款以視光元素為主題的反應戰鬥遊戲。核心樂趣是看清視標方向、快速滑動斬擊，長期目標是加入 PVP、PVE、天梯與答題養成，讓玩家透過「讀書答題」解鎖或提升招式、血量、角色能力。

風格可以保留搞笑與誇張演出，但視光元素要穩定存在，例如視力表方向判讀、白內障、散光、散瞳、青光眼、黃斑部等概念。建議避免讓醫療名稱變成錯誤知識，招式可以中二，但題庫與說明要準確。

## 推薦技術方向

目前最適合先用 Capacitor 包 Web App。

理由：

- 現有原型就是 HTML/CSS/JavaScript，轉移成本最低。
- 你是程式初學者，Capacitor 比 Swift/SpriteKit 或 React Native 重寫更容易接上現有成果。
- 後續仍可加入 iOS 原生能力，例如震動、音效控制、登入、推播、內購。
- 真正困難的部分不是畫面框架，而是 PVP 同步、帳號、天梯、防作弊與題庫管理，這些不會因為重寫成原生就消失。

建議堆疊：

- 前端：Vue 3 + Vite + TypeScript
- App 包裝：Capacitor
- 動畫：GSAP 可保留
- 狀態管理：Pinia
- 後端：Node.js / NestJS 或 Fastify
- 即時對戰：WebSocket
- 資料庫：PostgreSQL
- 登入：Apple Sign In 優先，之後可加 Google
- 排行榜與玩家資料：後端權威保存

## 對戰架構建議

PVP 不能只相信玩家手機送來的結果，否則很容易作弊。建議採「伺服器權威」：

- 手機只送玩家輸入，例如斬擊方向、時間戳、技能使用。
- 伺服器判定是否命中、傷害、MP、血量與勝負。
- 伺服器把戰鬥狀態同步給雙方。
- 客戶端負責做動畫與手感預測，但最終結果以伺服器為準。

第一版 PVP 可以先做「同步回合制即時戰鬥」，不要急著做完全格鬥遊戲等級的低延遲同步。Samureye 的玩法很適合把每 0.1 到 0.2 秒作為判定節奏，容忍一點網路延遲。

## 遊戲模式

第一階段保留 PVE：

- 玩家對 Master HOU 或其他電腦敵人。
- 用 PVE 打磨手感、技能、難度曲線。
- PVE 可作為新手教學與每日任務。

第二階段加入 PVP：

- 快速配對
- 好友房間
- 休閒對戰
- 排名對戰

第三階段加入天梯：

- 玩家分數、段位、勝率、連勝
- 每季重置
- 防止刷分與中離懲罰
- 排行榜可分總榜、好友榜、賽季榜

## 教育養成系統

核心規則可以是：戰鬥能力提升必須透過答題獲得。

可行設計：

- 答對視光題目獲得 Study Points。
- Study Points 用於解鎖技能、升級血量、降低技能 MP 成本、提升 PVE 關卡進度。
- PVP 排位中避免重課或高等級碾壓，建議用「標準化能力」或分模式處理。

建議分兩種對戰：

- 公平競技：所有玩家固定血量與技能等級，只看反應與策略。
- 養成對戰：允許讀書升級後的角色能力生效。

這樣能同時保住教育動機與競技公平。

## 裝置策略

第一版建議 iPhone 直式限定。

理由：

- 現在操作是單手/雙手直式滑動，直式最自然。
- 橫式會壓縮上下 HUD，視標、血條、技能按鈕都會變擠。
- 先做好 iPhone 安全區、不同螢幕高度、Dynamic Island、Home indicator，比同時支援橫式更重要。

iPad 建議第二階段處理：

- iPad 直式可以放大中心視標、增加左右資訊欄。
- iPad 橫式需要獨立版面，適合做觀戰、PVE 地圖或題庫頁，不一定適合戰鬥主玩法。

## 開發順序

### 0. 原型修補

- 補 iPhone 安全區。
- 補觸控狀態收尾。
- 補敵方命中統計。
- 修正第一回合時間。
- 先維持單檔 HTML，方便確認玩法。

### 1. 專案化

- 改成 Vue 3 + Vite 專案。
- 移除 CDN，所有依賴由 npm 管理。
- 拆分 GameView、HUD、SkillBar、ResultScreen、SoundEngine。
- 建立資料檔：skills、enemySkills、questions、stages。

### 2. iOS 包裝

- 加入 Capacitor。
- 鎖定 iPhone 直式。
- 設定 app icon、splash screen、安全區。
- 實機測試觸控、音效、效能與離線啟動。

### 3. PVE 完整化

- 關卡列表。
- 敵人技能差異。
- 獎勵結算。
- 每日題目與每日挑戰。

### 4. 教育系統

- 題庫後台格式。
- 答題頁。
- 答對獎勵。
- 技能/血量/角色能力升級。
- 錯題複習。

### 5. 帳號與雲端資料

- Apple Sign In。
- 玩家資料。
- 進度同步。
- 題目答題紀錄。

### 6. PVP 與天梯

- WebSocket 對戰房。
- 快速配對。
- 伺服器判定戰鬥。
- 排名積分。
- 賽季排行榜。

## 下一個最小可行版本

建議第一個可上 TestFlight 的版本只做：

- iPhone 直式 PVE
- 3 到 5 個敵人
- 20 到 50 題視光題庫
- 答題獲得 Study Points
- 3 個技能升級
- 本機排行榜或雲端簡易排行榜

PVP 可以在第二個 TestFlight 版本加入。這樣第一版能比較快拿給朋友測試，也能驗證「答題養成是否真的讓玩家想回來」。

## 待確認

- 題庫內容你希望偏專業證照、一般大眾衛教，還是遊戲化冷知識？
- PVP 是否要允許升級能力生效，或排位賽必須完全公平？
- 是否需要未成年玩家使用？這會影響隱私政策、題材表述、帳號與資料保存。
- 遊戲是否會有付費內容？若有，iOS 需要走 Apple 內購規則。
